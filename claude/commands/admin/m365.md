<objective>
Manage Microsoft 365 tenant using the CLI for Microsoft 365 MCP server and the m365 CLI. This skill provides persistent knowledge so Claude does not need to re-learn M365 administration each session.

Route based on argument:
- **No argument / interactive**: Ask what the user wants to do
- **login**: Authenticate to Microsoft 365 tenant
- **status**: Show current login status and tenant info
- **users**: User management (list, create, update, delete, licenses)
- **groups**: Group management (M365 groups, security groups, distribution lists)
- **sharepoint**: SharePoint admin (sites, lists, permissions)
- **teams**: Teams admin (teams, channels, apps)
- **entra**: Entra ID admin (apps, service principals, admin units)

## MCP Server: m365 (@pnp/cli-microsoft365-mcp-server)

Official MCP server from the PnP community. Provides natural language management of Microsoft 365 through 3 core tools.

### Prerequisites
1. **Node.js 20.x+** must be installed
2. **CLI for Microsoft 365** installed globally: `npm i -g @pnp/cli-microsoft365` (already installed)
3. **Must be logged in first**: Run `m365 login` before using MCP tools

### Available MCP Tools

| Tool | Purpose |
|------|---------|
| `m365SearchCommands` | Fuzzy search CLI commands by query string — use this to discover commands |
| `m365GetCommandDocs` | Get detailed docs and examples for a specific command |
| `m365RunCommand` | Execute a CLI for Microsoft 365 command and return results |

### How to Use the MCP Tools Effectively

**Step 1 — Find the command:**
Use `m365SearchCommands` with a natural language query like "list users" or "create SharePoint site"

**Step 2 — Get the docs:**
Use `m365GetCommandDocs` with the exact command name to see parameters and examples

**Step 3 — Run the command:**
Use `m365RunCommand` with the full command string

## Common Command Reference

### Authentication
```bash
m365 login                           # Interactive browser login
m365 login --authType deviceCode     # Device code flow (headless)
m365 login --authType certificate --certificateFile ./cert.pem --appId <id>  # App-only
m365 status                          # Check current login status
m365 logout                          # Log out
```

### User Management
```bash
m365 entra user list                           # List all users
m365 entra user get --id user@domain.com       # Get user details
m365 entra user add --displayName "John Doe" --userName john@domain.com --accountEnabled true --mailNickname john --password "TempPass1!"
m365 entra user set --id user@domain.com --jobTitle "Developer"
m365 entra user remove --id user@domain.com    # Remove a user
m365 entra user license list --userName user@domain.com  # List licenses
m365 entra user license add --userName user@domain.com --ids <sku-id>  # Assign license
```

### Group Management
```bash
m365 entra m365group list                      # List M365 groups
m365 entra m365group get --id <group-id>       # Get group details
m365 entra m365group add --displayName "Dev Team" --mailNickname devteam
m365 entra m365group user list --groupId <id>  # List group members
m365 entra m365group user add --groupId <id> --userName user@domain.com
m365 entra group list                          # List all groups (security + M365)
```

### SharePoint Administration
```bash
m365 spo site list                             # List all sites
m365 spo site get --url https://tenant.sharepoint.com/sites/mysite
m365 spo site add --type CommunicationSite --title "Alpha Testing" --url https://tenant.sharepoint.com/sites/alpha
m365 spo site remove --url https://tenant.sharepoint.com/sites/old
m365 spo list list --webUrl https://tenant.sharepoint.com/sites/mysite
m365 spo listitem list --listTitle "Documents" --webUrl https://tenant.sharepoint.com/sites/mysite
m365 spo storageentity list --appCatalogUrl https://tenant.sharepoint.com/sites/appcatalog
```

### Teams Administration
```bash
m365 teams team list                           # List all teams
m365 teams team get --id <team-id>             # Get team details
m365 teams team add --name "Alpha Testers" --description "Alpha testing team"
m365 teams channel list --teamId <team-id>     # List channels
m365 teams channel add --teamId <team-id> --name "alpha-bugs"
m365 teams user list --teamId <team-id>        # List team members
m365 teams user add --teamId <team-id> --userName user@domain.com --role member
m365 teams app list                            # List installed apps
```

### Entra ID (Azure AD)
```bash
m365 entra app list                            # List app registrations
m365 entra app get --appId <app-id>            # Get app details
m365 entra app add --name "MyApp Alpha"        # Register new app
m365 entra sp list                             # List service principals
m365 entra administrativeunit list             # List admin units
m365 entra administrativeunit member list --id <unit-id>
```

### Power Platform
```bash
m365 pp environment list                       # List Power Platform environments
m365 pa app list                               # List Power Apps
m365 flow list --environmentName <env>         # List Power Automate flows
m365 flow run list --environmentName <env> --flowName <flow>
```

### Tenant & Settings
```bash
m365 tenant id get                             # Get tenant ID
m365 tenant serviceannouncement health list    # Service health
m365 tenant serviceannouncement message list   # Message center posts
```

## PowerShell Fallback

For operations not covered by the m365 CLI, use `mcp__powershell__run_powershell`:

```powershell
# Connect to Exchange Online
Connect-ExchangeOnline -UserPrincipalName admin@domain.com

# Connect to Microsoft Graph
Connect-MgGraph -Scopes "User.Read.All","Group.ReadWrite.All"
Get-MgUser -All | Select-Object DisplayName, UserPrincipalName, AccountEnabled
```

<process>
1. Parse the user's argument to determine the operation category
2. If no argument, ask what they want to do using AskUserQuestion
3. Check login status first — if not logged in, guide through `m365 login`
4. Use `m365SearchCommands` to find the right command if unsure
5. Use `m365GetCommandDocs` to get exact parameter names and examples
6. Use `m365RunCommand` to execute the command
7. For complex operations requiring multiple steps, chain commands sequentially
8. Always confirm before destructive operations (remove user, delete site, remove group)
</process>

<critical_rules>
- NEVER remove a user, group, or site without explicit user confirmation
- ALWAYS check login status before running commands — guide to `m365 login` if needed
- NEVER expose or log passwords, tokens, or certificates in output
- For bulk operations (adding many users, assigning licenses), show the plan first and confirm
- When searching for commands, prefer `m365SearchCommands` over guessing command syntax
- If a command fails, check `m365GetCommandDocs` for correct syntax before retrying
- Output results in clean formatted tables when listing items
- For tenant-wide changes (policies, settings), always confirm impact before proceeding
</critical_rules>
